npm 安装包权限错误 /usr/lib/node_modules

2020-03-28
config error

文章目录

当你使用 npm global 安装npm包时候，例如

npm install -g handlebar

这时就有可能发生错误

[Error: EACCES: permission denied, access ‘/usr/lib/node_modules’]

这是因为普通用户没有/usr 目录下的写入权限。当然你可以使用sudo，但是这并不是一个好方法。

sudo npm install -g handlebar

实际上npm不需要root权限，也不该给，一旦成百上千个依赖包里边有一个恶意的Script，估计电脑就要变肉鸡了。有很多文章讲这个Don’t Use sudo with npm, Do not sudo npm

解决方法

解决方法就是通过配置 npmrc 到当前用户下。把包安装到 ~/.npm_packages 类似于 pip install --user 。

第一步

首先将以下内容添加到 ~/.npmrc 下：

    prefix = ${HOME}/.npm_packages

或者执行

echo 'prefix = ${HOME}/.npm_packages' > ~/.npmrc

第二步

然后将 ${HOME}/.npm_packages/bin 添加到环境变量 PATH 里，将以下文本复制到~/.zshrc或者~/.bashrc 取决于你用哪一个。

    export PATH="${PATH}:${HOME}/.npm_packages/bin"

当环境变量更新后，就可以使用 npm -g install package 包就会被安装到 ~/.npm_packages/lib 下面了。
